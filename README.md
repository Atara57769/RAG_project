# RAG Project  Event-Driven RAG Chat

A Retrieval-Augmented Generation system that routes user questions through an event-driven workflow using structured JSON data and semantic search. This version uses an explicit `RAGWorkflow` with steps for routing, structured query fallback, retrieval, post-processing, and response synthesis.

## High-Level Architecture

```
User Question
      
app/chat.py -> RagService -> RAGWorkflow (domain/workflow.py)
      
Router (config/route_config.py)
       structured_data_query_engine (data/structured_data.json)
       semantic retrieval (Pinecone vector store)
      
Answer generated by LLM
```

## Project Structure

- `main.py`  launch file: draws workflow visualization and starts Gradio chat
- `app/chat.py`  Gradio chat interface and async `respond`
- `services/rag_service.py`  run and trace workflow execution
- `domain/workflow.py`  step-based event-driven workflow
- `domain/events.py`  custom events for workflow transitions
- `domain/errors.py`  custom domain exceptions
- `config/rag_config.py`  LLM, embeddings, Pinecone vector index, retriever, response synthesizer
- `config/route_config.py`  selector + structured query engine
- `data/load_data.py`  build vector index from docs
- `data/extract_structured_data.py`  extract decisions/rules/warnings/dependencies
- `data/structured_data.json`  structured knowledge store
- `docs/`  input docs to index and extract from
- `workflow_visualizations/`  generated HTML workflow traces
- `tests/`  unit tests

## Quick Start

1. Install dependencies:

```bash
pip install -r requirements.txt
```

2. Add `.env` keys:

```env
GROQ_API_KEY=...
COHERE_API_KEY=...
PINECONE_API_KEY=...
```

3. Add docs to `docs/`.

4. Build indexes and extract structured data:

```bash
python data/load_data.py
python data/extract_structured_data.py
```

5. Start app:

```bash
python main.py
```

Then open the Gradio URL in browser.

## Routing Flow

- `router` step uses `selector.select(choices, query=...)`.
- If selection index 0  structured mode (`structured_data_query_engine`).
- Else  semantic retrieval mode (vector search + postprocessor + response synthesizer).
- If structured answer says `NOT_FOUND`  fallback to retrieval.

## Commands

- Run tests:

```bash
python -m unittest discover -s tests
```

- Rebuild semantic index:

```bash
python data/load_data.py
```

- Rebuild structured JSON:

```bash
python data/extract_structured_data.py
```

## Notes

- Keep `data/structured_data.json` updated when docs change.
- Use `workflow_visualizations/last_execution.html` to inspect the latest workflow execution trace.
